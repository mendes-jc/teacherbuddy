# TeacherBuddy Phase 1 (MVP) Implementation Plan

> **For agentic workers:** implement task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** A beautiful, modern personal app to manage students and capture lesson notes with a per-student progress timeline — no AI yet.

**Architecture:** Next.js (App Router, TypeScript) with server components/actions talking to Supabase (Postgres + Auth). Row-Level Security scopes all data to the signed-in teacher. UI built with Tailwind + shadcn/ui.

**Tech Stack:** Next.js 15, TypeScript, Tailwind CSS, shadcn/ui, Supabase (`@supabase/ssr`), Recharts (progress chart), Zod + react-hook-form (forms).

---

## File structure

```
app/
  layout.tsx                 # root layout, fonts, theme
  globals.css                # tailwind + shadcn theme tokens
  page.tsx                   # redirect → /dashboard or /login
  login/page.tsx             # email/password auth
  auth/callback/route.ts     # supabase auth code exchange
  (app)/layout.tsx           # authed shell (sidebar/topbar)
  (app)/dashboard/page.tsx   # student list + add student
  (app)/students/[id]/page.tsx  # student detail (profile, chart, timeline, add note)
components/
  ui/*                       # shadcn primitives
  app-shell.tsx              # sidebar + header
  student-card.tsx
  student-form.tsx           # create/edit student (dialog)
  lesson-note-form.tsx       # add lesson note (dialog)
  lesson-note-item.tsx       # one timeline entry
  progress-chart.tsx         # recharts line of progress_rating
  struggle-tags-input.tsx    # tag input
  empty-state.tsx
lib/
  supabase/server.ts         # server client (cookies)
  supabase/client.ts         # browser client
  supabase/middleware.ts     # session refresh
  types.ts                   # DB row types (Student, LessonNote)
  students.ts                # data access: list/get/create/update students
  lesson-notes.ts            # data access: list/create lesson notes
  actions.ts                 # server actions wrapping data access + revalidate
middleware.ts                # route protection
supabase/migrations/0001_init.sql  # schema + RLS
.env.local.example
```

## Verification notes

- `npm run build` must pass (type-check + compile). Dynamic pages use `export const dynamic = "force-dynamic"` so build doesn't require a live DB.
- Full runtime verification (login, CRUD) requires the user's own Supabase project URL + anon key and running `0001_init.sql`. This is documented in the README; it cannot be exercised end-to-end without those credentials.
